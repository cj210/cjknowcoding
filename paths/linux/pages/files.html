Useful commands to work with files:

Commands for handling files
Command Options Arguments Usage Example
less File name Tde less command displays tde contents of a file, one page at a time. 
cj:testDir/ $ less file.txt
tac File name tac (cat spelled backwards) will concatenate and print files in reverse.

head Filename The head command prints a portion of a file, starting from the beginning of the file. By default, it prints the first 10 lines of a file. 
~/testDir❯ head file.txt 
Red
Green
Yellow
blue
Gold
apple
Android
rain
yesterday
-n --lines integer (number of lines) Filename Prints first n lines of file Filename 
~/testDir❯ head -n 3 file.txt 
Red
Green
Yellow
~/testDir❯ head -2 file.txt 
Red
Green
-c Number of bytes, File name Prints bytes instead of lines 
~/testDir❯ head -c 6 file.txt
Red
Gr%
tail -n --lines -c Number File name The tail command works similarly to the head command, except it prints from the END of a file. By default, it prints the last 10 lines of a file. 
~/testDir❯ tail file.txt 
Red
Green
Yellow
blue
Gold
apple

~/testDir❯ tail -3 file.txt

~/testDir❯ tail -c 12 file.txt 
n
yesterday
wc -l -w File name The word count command can tell us the number of words, lines, or bytes in files. By default, it prints out three numbers: the lines, words, and bytes in a file 
~/testDir❯ wc file.txt 
 9  9 56 file.txt
~/testDir❯ wc -l file.txt 
9 file.txt
~/testDir❯ wc -w file.txt 
9 file.txt
sort -r -n -u -k File name The sort command outputs the sorted contents of a file (it does not change the file itself). By default, it will sort the lines of a file alphabetically. 
~/testDir❯ sort -r file.txt
