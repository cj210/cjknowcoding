The linux directory structure:

Common Linux Directories
Directory Usage
/ Root . Everything starts at root.
/bin Binary files and other executable programs are stored here. Programs are written in source code which are in human readable text. These text files are compiled into machine readable binaries i.e., 0s and 1s. The applications that we run are stored here.
/etc System configuration files that controls the OS and other applications behavior are stored here.
/home User directoriesare stored here for all the users.
/opt Optional 3rd party scripts that are not bundled with the operating system are found here.
/tmp This is a temporary space which is cleared on reboot.
/usr User selected programs live here.
/var Contains files / data that often change. ex: log files
/boot Files that are required to start or reboot OS, including the Kernel, are present here.
/dev Device files, which are controlled by OS and system administrators are present here.
/cgroup The control group hierarchy.
/lib The system libraries.
/export The shared file system.
/sbin System administration binaries.
/proc Provides information on running processes.
Note:
  • All the above directories are the top level directories.
  • The applications installed by user mostly reside either in /usr or /opt.
  • For applications installed, there can be sub-directories with the same convention and the directories have the same usage. Difference is that the scope is limited to that particular application. ex: /usr/local/appName/bin directory has the binaries for the particular application, "appName".
