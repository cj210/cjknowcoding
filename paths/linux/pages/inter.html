tar [-] c | x | t f filename sourceDirectory

File compression:

Commands for file size and compression
Command Options Usage
du Estimates file usage
-k Displays size in kile bytes
-h Displays sized in human readable form.
gzip Compress files
gunzip Uncompress files
zcat gzcat Both commands concatenate or diplay content. (similar to cat command) 
~/testDir/salads/fruits❯ du apple 
4	apple
~/testDir/salads/fruits❯ du -k apple 
4	apple
~/testDir/salads/fruits❯ du -h apple 
4.0K	apple
~/testDir/salads/fruits❯ gzip apple 
~/testDir/salads/fruits❯ du -h apple.gz 
4.0K	apple.gz
~/testDir/salads/fruits❯ gunzip apple

Expansion and substitution:

Wildcard charectors / Globbing patterns:

We can use special wildcard characters to build patterns that can match multiple filenames at once. The asterisk ( * ) character represents zero or more characters in a filename. The question mark ( ? ) character represents any single character.

Note: To match a wildcard charector, use an escape charector \.

Range Wildcards:

Inside of square brackets ([ ]) we can specify a range of characters to match.

~/testDir❯ ls *[A-D].rod
beanA.rod  beanB.rod  beanC.rod  beanD.rod
~/testDir❯ ls *[A-D].rod *[5-7].png pic[148].*
beanA.rod  beanB.rod  beanC.rod  beanD.rod  pic1.png  pic4.png  pic5.png  pic6.png  pic7.png  pic8.png

Negative ranges:

Inside of square brackets ([ ]) we can specify a range of characters to NOT match, using a caret ( ^ )

~/testDir❯ ls *[^A-D].rod                         
beanE.rod  beanF.rod  beanG.rod
~/testDir❯ ls pic[^1348].*                        
pic2.png  pic5.png  pic6.png  pic7.png  pic9.png
~/testDir❯ ls *[^3-7].png                         
pic1.png  pic2.png  pic8.png  pic9.png

Charector classes:

  • [:alpha:] - alphabetic characters, upper and lower
  • [:digit:] - digits -1-9
  • [:lower:] - lower case letters
  • [:upper:] - upper case letters
  • [:blank:] - blank characters: space and tab
  • [:punct:] - punctuation characters
  • [:alnum:] - alphanumeric characters (alpha + digit)

Brace expansion:

Brace expansion is used to generate arbitrary strings. Basically, it will generate multiple strings for us based on a pattern. We provide a set of strings inside of curly braces ({ }), as well as optional surrounding prefixes and suffixes. The specified strings are used to generate all possible combinations with the optional prefixes and suffixes

~/testDir❯ mkdir week{1..5}
~/testDir❯ echo {1..50..5}
1 6 11 16 21 26 31 36 41 46
~/testDir❯ touch Group{A..F}

Arithmetic expansion:

The shell will perform arithmetic via expansion using the $((expression)) syntax. inside the parentheses, we can write artithmetic expressions using:

  • + addition
  • - subtraction
  • * multiplication
  • / division
  • ** exponentiation
  • % modulo (remainder operator)

~/testDir❯ echo $((3+45))
48
~/testDir❯ echo $((3*5)) 
15
~/testDir❯ echo $((3**5))
243

Note: The shell only performs integer arithmetic, so the result is always a whole number.

Command substitution:

We can use the $(command) syntax to display the output of another command.

~/testDir❯ echo "Today is $(date)"            
Today is Sunday 23 July 2023 03:56:40 PM IST

Quoting:

  • If we wrap text in double quotes, the shell will respect our spacing and will ignore special characters except for dollar sign ( $ ) , backslash ( \ ), and backtick ( ` ) Double Quotes. Pathname expansion, brace expansion, and word splitting will be ignored. However, command substitution and arithmetic expansion is still performed because dollar signs still have meaning inside double quotes.
  • Use single quotes to supress all forms of substitution.
  • To selectively prevent expansion or substitution for specific charectors, we can prefix them with a single backlash \. We can use this to reference special charectors that normally have meaning inside of filenames.
~/testDir❯ echo This is too                                    far
This is too far
~/testDir❯ echo "This is too                    far"
This is too                    far
~/testDir❯ echo "$hit creature"
 creature
~/testDir❯ echo '$hit creature'
$hit creature
~/testDir❯ echo $500.45
.45
~/testDir❯ echo \$500.45
$500.45
