Customising the prompt:

Prompt can be customised using the environment variables. For bash, it is $PS1.

    export PS1=[u\@h\w\]\$
    Add the above line to .bashrc file.
    For multiple lines add few outside the []. like
    PS1="\t\n[\h\w]\$"
    Use man bash and search for PS1 for more details.

The Environment:

The shell maintains a set of information during a shell session, known as the environment. It's just a series of key-value pairs that define properties like:

  • Your home directory
  • Your working directory
  • The name of your shell
  • The name of the logged in user

Key points:

  • printenv - Command prints all the environmental variables.
  • Environment variables can be set and updated as VAR="value"
  • Environmental variable can be deleted by unset command unset VAR
  • Environmental variable can be retained through sessions by adding value to .bashrc file. export VAR="value"

Processes:

Key points:

  • command &
     - Starts the command in background
  • Ctrl-c
     - Kills the foreground process. 
  • Ctrl-z
     - Suspends the foreground process.
  • bg [%num]
    - Background a suspended 
  • fg [%num]
     - Foreground a background process.
  • job [%num]
     - List jobs.
  • kill
     - Kill a process by job number or PID.
  • kill -l
     - Display a list of signals.

Cron

  • cron - A time based job scheduling service.
  • crontab - A program to create, read, update, and delete your job schedules.
  • cron is used to schedule and automate tasks.
  • crontab file Install a new crontab from file.
  • crontab -l List your cron jobs.
  • crontab -e Edit your cron jobs.
  • crontab -r Remove all of your cron jobs.
Examples:
# Run every 30 minutes.
0,30 * * * * /opt/acme/bin/half-hour-check

# Another way to do the same thing.
*/2 * * * * /opt/acme/bin/half-hour-check

# Run for the first 5 minutes of the hour
0-4 * * * * /opt/acme/bin/first-five-mins

# Run at 02:00 every day and
# send output to a log file.
0 2 * * * /root/backupdb > /tmp/db.log 2>&1
