    Getting started with git:

    Install Git. Then run the following commands in CLI to set configurations.

    • Setting up local system
      •             git config --global user.email "your@email.id"
            git config --global user.name "your name"
    • Set your prefered editor
      •             git config --global core.editor vim
    • Verify the changes made
      •             git config --global --list

    Getting started with github:

    Repositories or the repos can be created in local system CLI and also in github.The simplest method is to create repo in github and then clone it into local system. This can be done as follows

    • Create an access token in developer settings < classic tokens
    • Create a new repo
    • Copy the web URL of the newly created repo
    • In the local system CLI, use the command
      •             git clone https://accesstoken@remainingPartOfWebURL
    • Once cloning is done, to update changes use
      •             git pull
    • After making changes to repo, use
      •             git status
            git add .
            git status
            git commit -m "any commit message you wish"
            git push -u origin main

    The Flow:

    Initiating the project:

    To work with git and github, you need some files for which the version control can be applied. Lets consider different possibilities of initiating the project.

    1. Starting from scratch:
      2.             git init project-name
    2. Existing project:
      3.             #Get in to the main directory of the project. Use command
            git init
    3. Forking and cloning: Once the cloning is done, the flow is same as mentioned for "Getting started with github".

    Tracking files:

    • Files in and not in staging area can be checked by
      •             git status
    • To add files into staging area
      •             git add filename
            
            #To add all the changed files
            git add .
    • To list the files that git is tracking
      •             git ls-files
    • Files, once in the staging area can be commited using
      •             git commit -m "Any apt message"
    • To commit the files already being tracked, with out adding to staged area
      •             git commit -am "any apt message"
            Note: This doesnt include newly created files, not staged atleast once.

    Backing out files:

    • You have made the changes, moved to staging area and then want to unstage.
      •             git reset HEAD filename
    • To completely discard changes in a unstaged file.
      •             git checkout -- filename

    Renaming and moving files:

    When a file is renamed, at OS level, git considers it as a file delted and new file added. And it wont be in staging area. This adds another step of moving of files in to staging area using git add filename. To avoid this, we can rename files using git.

                git mv oldname.txt newname.txt

    Deleting files

    Similar to renaming, files can also be deleted using git and save an extra step using

                git rm filename
            # -r and -f options apply.
            Note: Changes made before commit wont be saved.

    History:

    git log
    command is used to display the commit history. Some important options are:

    • git log --abbrev-commit
       - Displays all commits with a shorter commit ID 
    • git log --oneline --graph --decorate
       - More human readable form. 
    • git log abbreCommit1...abbreCommit2
       - Displays commits between the abbreviated commit IDs provided (range). 
    • git log --since="3 days ago"
       - Displays commits from the last 3 days. 3 can be any number. 
    • git log -- filename
       - Displays commits specific to file with name filename. 
    • git log --follow -- filename
       - Follows renames / changes for a specific file with name filename. 
    • git show fullcommitID
       - Shows complete information on the particular commit.

    The .gitignore file:

    Any file that is not required to be be tracked can be added to .gitignore file
